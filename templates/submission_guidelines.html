Please ensure that your JSON input follows the same format when making requests to the test server.

Input

We provide a list of test videos, along with their activity label and candidate steps. You can find the input file here. The video IDs are the original YouTube IDs used in HowTo100M. For each step, we provide the 'step_headline' and 'step_paragraph' entries, which can be used to ground the step.

The test server accepts JSON input containing video IDs as keys and lists of single timestamps as values.

The timestamps in the JSON represent your model's predictions for grounding the step at the corredponsing index in the list. The order of the steps aligns with the provided input file, which lists the candidate steps for each test video.

    {

  • Video ID "---0tKA3iYI" has timestamps: [196, 456, 0, 105, 721, 82]
  • Video ID "--Gscx86GA0" has timestamps: [0, 295, 0, 143, 612]
  • Video ID "--KXjWTy7Yw" has timestamps: [200, 48, 831, 0, 402]
Predictions for the non-alignable steps are ignored.
